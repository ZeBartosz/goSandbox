Go concurrency failure modes

Use this while fixing exercises/mixed_debugging_lab_test.go. For each bug, ask: compile, run, block, panic, race, leak, or wrong value?

unbuffered channel

Send before receiver

send 42
needs
receiver now

An unbuffered channel has no storage. The sender and receiver must rendezvous.

ch := make(chan int)
ch <- 42 // blocks
v := <-ch // never reached
Fix by starting a receiver/sender goroutine, or by reordering so a matching operation can happen.
buffered channel

Buffer full before receive

10
full
then
send 20 blocks

A buffered channel stores up to capacity. A send to a full buffer blocks immediately.

ch := make(chan int, 1)
ch <- 10
ch <- 20 // blocks before receives
Fix by receiving before the second send, increasing capacity intentionally, or using another goroutine.
closed channel

Receive vs send

Receiving from a closed empty channel is allowed. Sending to a closed channel panics.

v, ok := <-ch // ok false when closed + empty
ch <- 2       // panic if ch is closed
ok == false is the signal. The zero value alone is not enough.
range over channel

Never closed

1
2
then
wait forever

for v := range ch ends only when the channel is closed.

go func() {
  out <- 1
  out <- 2
  // forgot close(out)
}()
Usually the sending side closes the channel when no more values will be sent.
WaitGroup

Counter mismatch

Add(2)
→
Done()
→
counter 1

Wait blocks until the counter is zero. Every added task needs a matching Done.

wg.Add(2)
go func() { defer wg.Done() }()
wg.Wait() // blocks
Make Add match the number of goroutines/tasks, or start the missing worker.
shared state

Race on counter

count++ is read, add, write. Two goroutines can interleave and lose updates.

// protect the critical section
mu.Lock()
count++
mu.Unlock()
Run go test -race. Use a mutex for shared state critical sections.
atomic

Load + Store is not increment

Each operation is atomic, but the pair can interleave.

old := count.Load()
count.Store(old + 1) // wrong increment

count.Add(1)         // correct increment
Atomic means the whole operation cannot be interleaved. Choose the operation that matches your intent.
