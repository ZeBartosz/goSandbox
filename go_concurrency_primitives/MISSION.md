# Mission: Go concurrency primitives

## Why
Learn Go's core concurrency primitives to perform better in Go role interviews and to become confident building personal Go projects that do work safely in parallel.

## Success looks like
- Explain when to use goroutines, channels, mutexes, atomics, and wait groups.
- Debug simple deadlocks, races, blocked sends/receives, and goroutine leaks.
- Implement small concurrent patterns such as fan-out/fan-in and worker pools.
- Recognise when concurrency is unnecessary and a sequential solution is clearer.

## Constraints
- Current level: new to Go, comfortable with basic syntax.
- Preferred learning style: visual HTML explainers, debugging broken examples, short explanations, and coding exercises.
- Use the user's compiler/database learning projects as context where useful.

## Out of scope
- Distributed systems, advanced scheduler internals, and production performance tuning until the primitives feel solid.
